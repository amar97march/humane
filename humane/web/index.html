◉ Overview

Your agent's live state at a glance. The six dimensions (energy, mood, fatigue, boredom, social load, focus) drive every decision the agent makes.

How to use this page
  • DQM Score — Decision Quality Multiplier. Scales every action's confidence. Low DQM = more actions held for review.
  • Evaluate Action — Test how the 10-engine gate stack judges any action. Enter a type, confidence, and rationale to see the verdict.
  • Fire Impulse — Manually trigger a proactive impulse. Normally these fire automatically at random intervals.
  • Customize — Drag-and-drop widgets to reorder, or hide sections you don't need.
Example scenario
  • You notice DQM is 0.62 (low). Check the state bars — Fatigue is 0.85. That's why. The agent is "tired" and deferring risky actions.
  • Click Evaluate Action → type "send_proposal", confidence 0.7, rationale "Client requested it". You see the verdict is HOLD because fatigue pushed adjusted confidence below threshold.
  • You know this is urgent, so go to the Hold Queue and approve it manually. Or lower the Fatigue Defer Threshold in Settings.

♦ Entities

Every person or contact builds a relational profile. Trust, grudges, and sentiment shape how the agent communicates with each entity.

How to use this page
  • Trust Level — Derived from sentiment + grudge history. Ranges from Untrusted (human approval required) to Deep Trust (agent shares freely).
  • Sentiment — Weighted average of interaction quality. Decays over time — prospects decay fast (14 days), close colleagues slowly (60 days).
  • Grudge — Accumulated negative experiences. Decays at half the rate of positive sentiment — grudges linger, just like with humans.
  • Health — Overall relationship: Strong, Stable, Fragile, Strained, or Broken. Strained relationships trigger higher social risk.
Example scenario
  • You add a new client: click + Add Entity → name "Arjun", type "prospect". He starts with neutral trust and stable health.
  • After a positive call, the Telegram bot logs the interaction with sentiment +0.7. Arjun's trust starts moving toward trusted.
  • Two weeks later with no follow-up, sentiment decays (prospects decay in 14 days). The agent notices and sends you a reminder: "Haven't spoken to Arjun in 14 days."
  • If Arjun ghosts you, grudge builds up. The agent increases social risk on outgoing messages to Arjun — protecting you from seeming desperate.

⚑ Goals

Active objectives the agent tracks. Each goal has an ROI score — when it drops too low, the agent proposes abandonment. You always make the final call.

How to use this page
  • ROI — Return on Investment = velocity × relevance × (expected value / remaining effort). Green = healthy, amber = slowing, red = consider dropping.
  • Pause — Temporarily suspend a goal. It won't trigger abandonment proposals while paused.
  • Abandon — Mark as no longer worth pursuing. This is rational resource allocation, not failure.
  • Milestones — Set achievable milestone counts. Start with 3-5 per goal. Too many makes early ROI look artificially low.
Example scenario
  • Click + Add Goal → "Close DesignStudio deal", expected value 0.8, milestones 5. The goal starts with ROI 0.00 (red) — no progress yet.
  • After sending the proposal (milestone 1/5 done), ROI climbs to 0.35 (amber). You're on track.
  • Two weeks pass with no response. ROI drops to 0.12 (red). The agent sends you: "DesignStudio deal has stalled. ROI is 0.12 — want to abandon or push through?"
  • You click Pause while waiting for their response. The agent stops nudging you about it. When they reply, click Resume.

◇ Memories

The agent's knowledge base. Memories decay naturally — accessed ones get reinforced, forgotten ones get archived. Nothing is ever deleted.

How to use this page
  • Types — Episodic (events, fast decay), Semantic (facts, slow decay), Relational (feelings, very slow), Procedural (methods, near-permanent).
  • Relevance — Score 0-1. Below the retrieval threshold (default 0.3), memories get archived but still exist.
  • Pinned — Protected from decay permanently. Use for critical information the agent must never forget.
  • Search — Find memories by content. Toggle "Include Archived" to find forgotten items.
Example scenario
  • Click + Add Memory → "Priya's company uses React Native for mobile", type Semantic, pinned No. Relevance starts at 1.0.
  • Over the next month, if the agent never references Priya's tech stack, relevance decays to 0.25 and gets archived.
  • But if a conversation about mobile development happens and the agent retrieves this memory, relevance jumps back up — it got reinforced by access.
  • For critical facts like "Arjun's NDA expires March 2027", click Pinned: Yes. It will never decay, no matter how rarely accessed.

✦ Values

The agent's moral boundaries. Hard values block actions unconditionally — no override possible. Soft values flag actions for your review.

How to use this page
  • Hard Values — The absolute floor. An action that violates a hard value is blocked with no override path. Use for non-negotiable principles.
  • Soft Values — Flag for human review. You can approve a soft value violation when context justifies it.
  • Violation Examples — Concrete cases help the engine detect violations. More specific = better detection.
  • Presets — The "business-safe" preset includes 3 hard values and 4 soft values for professional contexts.
Example scenario
  • Click + Add Value → description "Never share client revenue data externally", severity Hard.
  • Add violation examples: "forwarding client P&L to a competitor, sharing revenue in a public forum".
  • Add honoring examples: "using anonymized data in case studies, getting written permission before sharing".
  • Now if the agent tries to draft an email containing revenue figures to a non-client entity, the Values Boundary Engine blocks it — no approval path, no override. The action never happens.
  • For softer guidelines like "avoid weekend messages", use severity Soft. The agent flags it for your review — you can override if it's truly urgent.

⚙ Settings

All tunable parameters for the 10 engines. Changes apply immediately — no restart needed.

How to use this page
  • Confidence Threshold — The most important setting. Lower it if too many actions are held. Raise it for higher-stakes environments.
  • Impulse Rate — Controls how often the agent acts on its own. Higher = more proactive behavior.
  • API Security — Enable authentication to require API keys for all requests. Generate and manage keys here.
  • Data Retention — Set auto-cleanup policies for old conversations, events, and resolved holds.
  • Feedback & Tuning — Export training data from approved/rejected actions. Auto-tune thresholds based on your patterns.
Example scenario
  • Your hold queue is always full — the agent holds everything for review. Go to Confidence Threshold (currently 0.65) and lower it to 0.50. Click Save. Immediately, more actions will auto-approve.
  • The agent keeps messaging contacts at midnight. Set Active Hours Start to 9 and End to 18. Impulses now only fire during business hours.
  • You want to integrate with Slack. Go to API Security → enable auth → click Generate Key. Copy the key (shown once). Use it in your Slack webhook: Authorization: Bearer hm_xxxxx.
  • After 3 months, click Feedback & Tuning → "Apply Recommendations". The system analyzes your approval patterns and suggests optimal thresholds.

✦ Agents

Run multiple independent agents from one dashboard. Each agent has its own personality, memory, goals, and values — completely isolated.

How to use this page
  • Create — Each new agent gets its own database, config, and conductor instance.
  • Switch — Click "Switch" on an agent card to route all dashboard actions through that agent.
  • Messages — Agents can communicate with each other via task handoffs, entity sharing, and alerts.
  • Isolation — Modifying one agent's state never affects another. They share the server, not data.
Example scenario
  • You run a design agency. Click + New Agent → name "Sales Bot", personality "assertive". Click again → "Support Bot", personality "empathetic".
  • Sales Bot tracks prospects, fires follow-up impulses aggressively (high impulse rate). Support Bot tracks client issues, is more cautious (high confidence threshold).
  • A prospect becomes a client. Click Share Entity on Sales Bot to copy "Arjun" to Support Bot. Now both agents know Arjun, but track him differently.
  • Sales Bot sends a message to Support Bot: "Arjun just signed — he needs onboarding." Support Bot picks it up in its inbox and starts the onboarding goal.
